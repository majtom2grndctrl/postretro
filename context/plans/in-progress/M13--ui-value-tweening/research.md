# Goal TW research notes — code anchors

Grounding for the spec, confirmed against source at draft time (2026-06). Ephemeral; do not maintain after ship.

## Current state (post-C)

- **Binds** — `crates/postretro/src/render/ui/descriptor.rs`: `TextBind { slot, format: Option<String> }`, `PanelBind { slot }`, both `Option<…>` on their widgets with `skip_serializing_if`. The tween sub-object extends these structs; widget structs themselves are untouched (D's territory).
- **Retained diff** — `render/ui/tree.rs` `resolve_bindings`: walks all node ids (`collect_node_ids`), resolves bound `Text`/`Panel` contexts against the frame's `slot_values`, compares against `last_resolved`, returns `BindingDiff { content_changed, appearance_changed }`. Text changes call `mark_dirty(node)` (taffy propagates to root); panel changes are appearance-only. `build_draw_data_retained` gates layout on `viewport_changed || content_changed || taffy.dirty(root)` and draw-list rebuild on `layout_recomputed || appearance_changed || content_changed || cache empty`; otherwise returns `cached_draw_data`. The tween driver slots into `resolve_bindings` and reuses this classification unchanged.
- **In-flight text measurement** — `measure_node` shapes `last_resolved.as_deref().unwrap_or(content)`; storing the eased display string in `last_resolved` + `mark_dirty` makes in-flight frames re-measure with no measure-seam change.
- **Value shapes** — `scripting/slot_table.rs` `SlotValue::{Number(f32-like), Boolean, String, Enum, Array(Vec<f32>)}`. C's `resolve_panel_fill` accepts `Array` of exactly 4; `slot_value_string` formats integral numbers without decimals (`42`, not `42.0`) — the in-flight rounded-integer rule renders through this same path.
- **Time** — `main.rs` owns `script_time: f64`, accumulated from `frame_dt` (line ~1140), already passed into `render_frame_indirect`. The snapshot publish site (`main.rs` ~1425–1431) builds `UiReadSnapshot::with_gameplay_tree(demo_tree, slot_values)` — the natural place to add the time argument. `StaticUiProxy` (`scripting/systems/ui_proxy.rs`) is the injected-dt precedent: deterministic, never wall-clock; its `flash_color_at` step-toggle (500 ms half-period, 3000 ms window) is what the demo's panel tween smooths.
- **Snapshot / signatures** — `render/ui/mod.rs` `UiReadSnapshot { version_line, gameplay_tree, slot_values }`; `UiPass::layout_gameplay_tree(tree, viewport, image_sizes, slot_values)` called from the gameplay record block (`render/mod.rs` ~4743–4755); `layout_tree` is the splash/fresh path (no retained state — inertness is structural).
- **Demo + gates** — `render/ui/demo.rs` `build_demo_descriptor` (health/ammo texts, flash swatch panel); `demo_ui_gate_test.rs` pins the appearance-only/relayout split and post-settle no-recompute — Task 4 updates these for the tweened frame sequence while preserving the post-settle guarantees.

## Design choices and their grounding

1. **Renderer-local display state, not store slots.** The roadmap left "UI-owned animated-value primitive and/or mod-facing slot writes" open. Slot-write tweening would put presentational state in the authoritative store, require an engine-side tweener writing through `write_store_slot` every frame, and force the `setState` decision early. Renderer-local state extends C's existing `last_resolved` per-node pattern (C's own Decision: per-node bound state lives in `NodeContext`, where "E and F add more per-node bound state that composes in the same place" — TW is exactly that).
2. **Snapshot-carried time over reusing `render_frame_indirect`'s `script_time` parameter.** The snapshot is the established once-per-frame game-logic→render contract (A); carrying time on it keeps `build_draw_data_retained` CPU-testable with synthetic times and keeps the splash path (which has its own render entry) trivially at `0.0`.
3. **Easing inlined, no crate.** Research §5: "Animation: inline or `keyframe`. Likely inlined." Four cubic curves are a dozen lines; no dependency earns its place.
