# Baked Static Direct SH for Entities + Billboards

> **Status:** draft.
> **Track:** Lighting — closes the half-built dynamic-object lighting model. Follows M9 (octahedral SH GI) and rides on M10's mesh pass.
> **Related:** `context/lib/rendering_pipeline.md` §4 (lighting), §8 (shader composition), §9 (skinned model pipeline) · `context/lib/experimental_spikes.md` (Task 0 validation harness convention) · `context/plans/in-progress/M10--mesh-render-pass/` · sibling `research.md` (code-grounding citations).

## Goal

Give moving objects (mesh entities, billboards) the room's baked direct brightness they currently lack. The engine bakes lighting in two disjoint parts: lightmaps carry DIRECT light on static surfaces; the octahedral SH volume carries INDIRECT (bounced) light only. Entities and billboards can't use lightmaps, so they are lit by indirect SH alone and read much dimmer than the lightmapped geometry around them. Add a SECOND, dense, baked SH layer carrying the DIRECT light arriving at each probe from STATIC lights (per-light, shadow-ray visibility, projected into L2 SH). Dynamic shaders sample indirect + direct and finally match the room — at no runtime light-loop cost, because it is baked.

Net runtime symmetry after this lands: static surface = lightmap-direct + SH-indirect; entity/billboard = SH-direct + SH-indirect.

## Scope

### In scope

- **Projection-convention validation harness (GATING FIRST STEP).** Before the dense bake or shader work, a one-probe / one-light A/B parity harness that bakes the direct-at-probe SH for a single probe under a single static light, reconstructs irradiance at a known normal, and asserts it matches the lightmap baker's direct value at an equivalent surface point/normal within a tolerance. This empirically confirms (or corrects) the D3 lobe form + normalization and is the gate for all downstream work.
- **Dense direct-at-probe SH bake.** For each probe, project the direct radiance from every STATIC light (the same `StaticBakedLights` set/order the lightmap baker uses), shadow-ray-visibility-tested, into L2 SH using the existing projection + cosine-lobe convolution. Dense (all probes), no animation.
- **Per-probe light-reach culling (in v1).** Before the per-probe shadow-ray pass, cull non-reaching static lights per probe (falloff-range + portal-reachability, analogous to the delta SH bake's affinity-cell clip), with the SAME determinism guarantees. Culling skips only provably-zero contributions, so baked output stays byte-identical to an unculled bake.
- **BC6H-at-rest compression of the direct atlas (in v1).** The direct atlas is stored BC6H-compressed at rest, reusing the same encoder/format the lightmap irradiance atlas already uses, and decoded by the hardware path at load. Octahedral tile geometry is unchanged (BC6H re-encodes only the texel payload), so the sampler math is untouched.
- **A SEPARATE direct atlas.** The direct data lives in its own octahedral atlas, tile geometry IDENTICAL to the indirect atlas. ONLY the dynamic shaders sample it. The shared indirect total atlas (`sh_total_atlas`) is unchanged.
- **New PRL section.** A new section id (mirroring `OctahedralShVolume`/`DeltaShVolumes`) carries the direct atlas. No `SH_VOLUME_VERSION` bump — existing v7 maps stay loadable (they simply lack the new section).
- **Entity + billboard shaders sample indirect + direct.** `skinned_mesh.wgsl` and `billboard.wgsl` add a direct atlas sample alongside the existing indirect sample and SUM the two irradiance terms before the albedo multiply.
- **Debug toolbar control.** A direct-scale slider + an isolation toggle (direct-only / indirect-only / combined) for the dynamic path, to verify brightness parity against lightmapped surfaces. Includes wiring a scale uniform into the mesh/billboard path (which today read no such field).
- **Bake caching + determinism parity.** The direct bake participates in the SAME build-stage cache and determinism contract as the indirect bake: byte-identical output, index-derived jitter, no RNG, no order-dependent reduction, cache-keyed on geometry + static-light set + probe layout.
- **Storage footprint reporting.** The dense direct atlas roughly doubles dense SH atlas bytes; this is measured and logged.

### Out of scope (non-goals)

- **Runtime group-2 dynamic-direct light loop.** The reserved mesh-pipeline group 2 ("dynamic-direct") for genuinely moving / script-spawned lights is a SEPARATE, later feature. This spec handles ONLY the baked static direct that entities are missing. Group 2 stays unallocated.
- **Sharp per-mover shadows.** Baked direct is intentionally soft and probe-resolution — that is the correct ceiling for movers under a baked term. Crisp dynamic shadows are the dynamic-direct/shadow-map track's job.
- **Animated static lights' direct.** Animated direct already lives in `lm_anim` (the animated weight-map bake). Folding animated lights into the direct-at-probe bake would double-count — animated lights are excluded here.
- **`forward.wgsl` / static-surface lighting changes.** Static surfaces keep lightmap-direct + SH-indirect, byte-for-byte. The static path is not touched.
- **A version-bump rebake of existing content.** No `SH_VOLUME_VERSION` change; old maps load unchanged.

## Decisions

**D1 — Direct lives in a SEPARATE atlas; the indirect total is untouched (LOAD-BEARING).** `sh_total_atlas` is the composed indirect total (base + animated delta) and is sampled by BOTH `forward.wgsl` (static) and the dynamic shaders. Folding direct into it would leak direct into the static path, double-counting against the lightmap — the exact failure the architecture forbids (`sh_bake.rs:34`, `delta_sh_bake.rs:8-12`). Direct gets its own atlas, sampled ONLY by the dynamic shaders. `forward.wgsl` is not modified; it stays indirect-only by not being pointed at the new data. "Static path unchanged / no double-count" is an acceptance criterion.

**D2 — The per-frame delta compose path is NOT the vehicle.** The delta layer is a sparse CSR-by-affinity-cell structure for ANIMATED lights, scaled by per-frame time curves in `sh_compose.wgsl`. The direct layer is STATIC (no curve) and DENSE (all probes). It does not belong in the per-frame delta CSR loop. Bake a standalone dense direct atlas and sample it directly — two full octahedral probe loops in the dynamic fragment (indirect + direct; each is the same 8-probe weighted SH evaluation, so the fragment runs TWO 8-probe loops, not two single fetches). (Alternative considered: a compose-time pre-sum into a dynamics-only "incident" atlas. Rejected as default — see Task 7; recommend the simpler two-loop approach unless a measured reason appears.)

**D3 — Probe-point direct projection convention (CENTRAL design decision; CONFIRMED-BY-HARNESS).** A probe is a point in air — there is no surface normal at bake time. "Direct at a probe" is therefore defined as: for each reaching static light, compute its incident radiance at the probe (the `light_contribution_lambert` falloff/intensity form, receiver = probe point) × `soft_visibility` shadow factor, then `accumulate_sh_rgb` that RGB value along the light's INCIDENT DIRECTION (probe→light unit vector) as a delta/cosine lobe; after all lights, `apply_cosine_lobe_rgb` so the stored coefficients are irradiance-convolved EXACTLY as the indirect bake stores them. At runtime the existing cosine-lobe convolution + the receiver fragment's own normal produce the per-fragment response — same sampler (`sample_sh_indirect_corners_depth_aware`), same math. Rationale: this is what makes the direct term reconstruct as cosine-weighted irradiance the receiver normal can read; getting the lobe direction or the normalization wrong is exactly what breaks brightness parity with the lightmapped room. **This convention is empirically gated by the Task 0 A/B parity harness (D7), which is the source of truth: if the harness shows the lobe form or normalization is wrong, D3's exact form is corrected from the harness findings before the dense bake + shader work proceed.**

**D4 — Same static-light set/order as the lightmap baker; static-only.** The direct-at-probe bake iterates the SAME `StaticBakedLights::from_lights` set in the SAME order the lightmap and indirect bakes consume, and ONLY static (`shadow_type == StaticLightMap`, non-animated, non-dynamic) lights. This keeps direct-at-probe parity with the lightmapped room and avoids re-introducing the animated double-count (`lm_anim` already owns animated direct).

**D5 — New section id (`SectionId::DirectShVolume`), no version bump; identical tile geometry.** Add `SectionId::DirectShVolume` (struct `DirectShVolumeSection`) mirroring `OctahedralShVolume = 34` / `DeltaShVolumes = 27`, rather than bumping `SH_VOLUME_VERSION` (7) — a version bump hard-rejects existing v7 `.prl` files and forces a full content rebake. The direct atlas uses `DEFAULT_IRRADIANCE_TILE_DIMENSION`/`DEFAULT_IRRADIANCE_TILE_BORDER` (the indirect tile geometry) so `sh_sample.wgsl`'s sampler math is reused unchanged. A map without the section degrades to direct = 0 (entities fall back to indirect-only, today's behavior).

**D6 — Cache + determinism are LOCKED contracts.** The direct bake honors the indirect bake's determinism rules: byte-identical output for identical inputs; soft-visibility jitter index-derived (`soft_visibility_seed`-style, perturbing sampling never geometry); no RNG; order-preserving fan-out (no float `reduce`, no HashMap-iteration to assemble output). It cache-keys on the same input kinds (geometry + static-light set + probe layout) and slots into the existing warm/cold driver paths. The light-reach culling (D9) lives INSIDE this contract — it narrows which shadow rays run without changing baked bytes — and BC6H-at-rest (D8) is the same documented determinism exemption the lightmap atlas already takes (`bc6h.rs:3`).

**D7 — Projection convention is validated FIRST by a one-probe/one-light A/B parity harness (GATING; LOAD-BEARING for D3).** Before any dense bake or shader work, a gating validation harness (using the harness/spike-test pattern from `experimental_spikes.md`; this is NOT a build-to-learn spike — it is a permanent regression test) bakes the direct-at-probe SH for ONE probe under ONE static light, reconstructs irradiance at a known normal, and LOGS the per-channel relative error against the lightmap baker's direct value at an equivalent surface point/normal (`light_contribution_and_direction` `lightmap_bake.rs:843` + `soft_visibility` `lightmap_bake.rs:1227`); the starting gate threshold is a few percent relative per channel, confirmed and pinned by the author from the first run. The harness is the gate for the dense bake (Task 2): it confirms — or corrects — the D3 lobe form + normalization before broad implementation. "The A/B parity harness passes within the pinned threshold" is an acceptance criterion; if it fails, D3 is updated from the harness findings (the harness is the source of truth, not D3's current wording).

**D8 — Direct atlas is BC6H-at-rest, mirroring the lightmap irradiance atlas (LOAD-BEARING for footprint; in v1).** The dense direct atlas roughly doubles dense SH atlas bytes, so it is stored BC6H-compressed at rest in v1 (not deferred), reusing the SAME encoder the lightmap irradiance atlas already uses — `bc6h::encode_bc6h_rgb_from_f32_rgba` (`bc6h.rs:71`), tagged `IRRADIANCE_FORMAT_BC6H` exactly as `CompositedAtlas::encode_section` (`lightmap_bake.rs:172`, BC6H route `:184`). Decode is the hardware `Bc6hRgbUfloat` path at load. BC6H re-encodes only the texel block payload — the octahedral TILE geometry (dim/border/atlas layout) is IDENTICAL to the indirect atlas (D5), so `sh_sample.wgsl`'s sampler math is untouched. The only new constraint is BC6H's ≥4 / 4-aligned dimension rule (`bc6h.rs:74`); the direct atlas dimension derivation must satisfy it (mirror the lightmap's power-of-two sizing or assert the alignment). A debug uncompressed bypass mirrors `uncompressed_irradiance` (`lightmap_bake.rs:103`) for A/B + determinism baselines. BC6H is the documented build-cache determinism exemption (`bc6h.rs:3`), and the encoder is itself deterministic, so the direct atlas inherits that determinism story (D6 still holds).

**D9 — Per-probe light-reach culling, mirroring the delta SH bake's affinity-cell clip (in v1; INSIDE the D6 determinism contract).** Direct adds a per-static-light shadow-ray pass at every probe; on light-dense maps this can dominate bake time. In v1 (not a follow-up) the direct bake culls non-reaching static lights per probe BEFORE the shadow-ray pass, using the SAME two-stage reach test the delta bake uses: (1) falloff-sphere AABB clip (`light_aabb` `affinity_grid.rs:274`, `cells_for_light` `affinity_grid.rs:194`) and (2) portal-reachability flood (`reachable_leaves` `affinity_grid.rs:156`), as assembled by `decompose_affinity` (`affinity_grid.rs:95`) and consumed by `bake_delta_sh_volumes` (`delta_sh_bake.rs:97`). A culled light is beyond falloff and/or not portal-reachable, so its `light_contribution_lambert` × `soft_visibility` is provably zero at that probe — culling skips only zero-contribution shadow-ray work and CANNOT change the baked coefficients. The cull set is derived purely from light geometry/origin + the static portal graph (index-derived, input-determined, no RNG); the bake fan-out stays order-preserving (no float reduce, no HashMap-iteration to assemble output), so the determinism + cache contract (D6) is unbroken. Baked output is byte-identical to an unculled bake; the cull report mirrors `log_per_light_culling` (`delta_sh_bake.rs:377`).

**D10 — Future group-2 dynamic-direct owns dynamic-tier lights ONLY; static-tier direct for movers is owned exclusively by this baked atlas (PINNED INVARIANT).** When the deferred group-2 dynamic-direct loop lands, it must evaluate ONLY dynamic-tier lights (`is_dynamic`) at runtime for movers — it must NOT re-evaluate the STATIC lights this baked direct atlas already covers, or a mover would receive static direct twice (baked SH + runtime loop). This is the same anti-double-count discipline the engine invariant requires (a light's contribution is never summed twice on one receiver) and mirrors the existing pins (`lm_anim` owns animated direct; the delta carries bounce only). The tier filter is the seam: this bake takes static-tier only (D4); group-2 takes dynamic-tier only. A future task implementing group-2 MUST honor this split; note it here so that work doesn't reintroduce the double-count.

## Acceptance criteria

### Visual / behavioral (eyeball in-engine, dev-tools)
- [ ] A mesh entity and a billboard placed in a directly-lit room read at a brightness that visibly MATCHES the lightmapped static geometry around them — not the dim indirect-only baseline they show today.
- [ ] An entity in a directly-lit cell is brighter than the same entity in a shadowed cell behind an occluder (baked shadow-ray visibility is present in the direct term).
- [ ] The debug toggle cycles direct-only / indirect-only / combined for the dynamic path; direct-only on a dynamic object reads as the room's direct brightness, indirect-only reads as today's dim baseline, combined sums them.

### Correctness / no-regression (test-gated where possible)
- [ ] `forward.wgsl` is unmodified and static-surface output is byte-identical to before this change (no direct double-count on lightmapped surfaces). A static surface's rendered color does not change.
- [ ] The shared indirect total atlas (`sh_total_atlas`) content is unchanged — the direct data is in a separate atlas/section.
- [ ] A map compiled WITHOUT the new section (legacy v7 `.prl`) loads and renders; dynamic objects fall back to indirect-only with no error. The loader does not reject it (no `SH_VOLUME_VERSION` change).
- [ ] Only static lights contribute to the direct atlas: a map whose only lights are animated produces a zero (or absent) direct section; no animated light's direct double-counts against `lm_anim`.

### Projection-convention gate (Task 0)
- [ ] The one-probe/one-light A/B parity harness passes: the harness LOGS the per-channel relative error between the runtime-reconstructed direct irradiance at a known normal and the lightmap baker's direct value at the equivalent surface point/normal. The starting threshold is a few percent relative per channel; the author confirms the exact value from the first run and pins it as the hard gate. (This gate confirms — or corrects — D3 BEFORE the dense bake and shader work; if it fails, D3 is updated from the harness findings.)

### Bake determinism / cache
- [ ] The direct bake produces byte-identical output across two runs on identical inputs (mirrors `sh_volume_bake_produces_byte_identical_output_on_repeated_runs`).
- [ ] A second compile of an unchanged map serves the direct section from the build-stage cache (cache hit, via the sibling cache entry keyed on geometry + static-light set + probe layout); changing geometry, the static-light set, or probe layout invalidates it and rebakes; the indirect bake's cache entry is untouched.
- [ ] Per-probe light-reach culling skips static lights beyond falloff / not portal-reachable BEFORE the shadow-ray pass, and the culled bake output is BYTE-IDENTICAL to an unculled bake (culling skips only provably-zero contributions, never changes baked values). The bake-time improvement (culled vs unculled) is measured and reported, mirroring `log_per_light_culling`.

### Format / footprint
- [ ] `SectionId::DirectShVolume` (struct `DirectShVolumeSection`) is registered and round-trips (encode → decode → equal); it does not reuse or collide with existing ids.
- [ ] The direct atlas is stored BC6H-compressed at rest (`IRRADIANCE_FORMAT_BC6H`, via `encode_bc6h_rgb_from_f32_rgba`) and decodes correctly at load via the hardware `Bc6hRgbUfloat` path; the octahedral tile geometry is unchanged so the sampler math is unaffected. The atlas dimensions satisfy BC6H's ≥4 / 4-aligned rule.
- [ ] The compiler logs the direct atlas byte footprint POST-BC6H-compression (and the dense pre-compression figure for comparison); the figures are reported in the PR alongside the indirect atlas size.

### Gates
- [ ] `skinned_mesh.wgsl` and `billboard.wgsl` pass naga validation.
- [ ] `cargo test` passes (workspace); `cargo clippy -- -D warnings` clean.

## Tasks

### Task 0 (GATING HARNESS): one-probe/one-light A/B projection-convention harness (`level-compiler` test)
Before any of Tasks 1–7, build a gating validation harness (using the harness/spike-test pattern from `experimental_spikes.md`; this is NOT a build-to-learn spike — it is a permanent regression test) that empirically validates the D3 projection convention. Bake the direct-at-probe SH for ONE probe under ONE static light using the D3 assembly (`light_contribution_lambert` receiver = probe, × `soft_visibility`, `accumulate_sh_rgb` along probe→light, then `apply_cosine_lobe_rgb`), reconstruct irradiance at a chosen normal via the runtime convolution, and LOG the per-channel relative error against the lightmap baker's direct value at an equivalent surface point/normal (`light_contribution_and_direction` `lightmap_bake.rs:843` + `soft_visibility` `lightmap_bake.rs:1227`). Keep the harness as a regression test so the convention can't silently drift. No dense bake, no shader edits until this passes.

### Task 1: New PRL section — direct octahedral atlas (`level-format`)
Add a new direct-SH section type `DirectShVolumeSection` (struct) with `SectionId` variant `DirectShVolume` in `crates/level-format/src` (a sibling to `sh_volume::OctahedralShVolumeSection`) carrying a dense per-probe octahedral irradiance atlas with tile geometry IDENTICAL to the indirect atlas (`DEFAULT_IRRADIANCE_TILE_DIMENSION`/`_BORDER`, same `irradiance_atlas_dimensions`/`tiles_per_row` derivation) plus the grid dims/origin/cell-size needed to bind the same `ShGridInfo`-shaped uniform. Register `SectionId::DirectShVolume` in `level-format/src/lib.rs` mirroring `OctahedralShVolume = 34` and `DeltaShVolumes = 27` (next free id, allocated at implementation time, no collision — AC 12 checks this; add the decode arm). Do NOT touch `SH_VOLUME_VERSION` (stays `7`). Provide `to_bytes`/`from_bytes` with a round-trip test, mirroring the existing section's encode/decode discipline. The Wire format section pins endianness/ordering. Because the runtime sampler (`sh_sample.wgsl`) is reused verbatim, the per-probe tile byte layout MUST match the indirect atlas tile exactly. The atlas payload is stored BC6H-compressed at rest (D8) with a format tag (`IRRADIANCE_FORMAT_BC6H`, plus the uncompressed-debug variant), so the section header carries the format tag exactly as the lightmap section does (`lightmap_bake.rs:177-191`); the BC6H encode itself happens at emit time (Task 3). Round-trip the format tag, and ensure the atlas dimensions satisfy BC6H's ≥4 / 4-aligned rule (`bc6h.rs:74`) — mirror the lightmap's power-of-two sizing or assert it.

### Task 2: Direct-at-probe SH bake + per-probe light-reach culling + cache integration (`level-compiler/src/sh_bake.rs`)
Add a direct-at-probe bake that, per probe, iterates the `StaticBakedLights` set (D4) and for each reaching static light: computes incident radiance at the probe (reuse the `light_contribution_lambert` falloff/intensity form with the probe as receiver), multiplies by `soft_visibility` (the SAME shadow-ray routine the lightmap + indirect bakes use, `segment_clear` for the hard case), and `accumulate_sh_rgb`s that RGB along the light's incident direction (probe→light) per the D3 convention (as gated by Task 0); after all lights, `apply_cosine_lobe_rgb`, then `pack_octahedral_irradiance_tile` into the Task 1 atlas. Reuse `ProbeGridLayout` so the direct probes land at byte-identical positions/validity to the indirect probes (no second grid).

**Per-probe light-reach culling (D9):** BEFORE the per-probe shadow-ray pass, cull non-reaching static lights using the SAME two-stage reach test the delta bake uses — falloff-sphere AABB clip (`light_aabb` `affinity_grid.rs:274`, `cells_for_light` `affinity_grid.rs:194`) + portal-reachability flood (`reachable_leaves` `affinity_grid.rs:156`), assembled by `decompose_affinity` (`affinity_grid.rs:95`) as `bake_delta_sh_volumes` (`delta_sh_bake.rs:97`) does. A culled light is provably zero-contribution at that probe, so the baked coefficients stay BYTE-IDENTICAL to an unculled bake. Add a test asserting culled-vs-unculled output equality, and a cull-savings log mirroring `log_per_light_culling` (`delta_sh_bake.rs:377`).

Honor the determinism contract (D6): the cull set is index-derived from light geometry/origin + the static portal graph (no RNG); index-derived jitter only (`soft_visibility_seed`-style); order-preserving `into_par_iter().map().collect()` fan-out, no HashMap-iteration to assemble output — add the byte-identical-output test mirroring `sh_volume_bake_produces_byte_identical_output_on_repeated_runs`. Slot into the build-stage cache the same way the indirect bake does, cache-keyed on geometry + static-light set + probe layout (the warm `sh_group`/cold `sh_bake` split at `main.rs:518-531` is the integration shape — the direct bake uses a SIBLING cache entry keyed on geometry + static-light set + probe layout, leaving the indirect bake's cache entry untouched). Excludes animated/dynamic lights at the filter, not downstream.

**Shadow-visibility test (AC 2):** Add a test using a two-probe scene (one probe in a directly-lit position, one probe occluded behind solid geometry): reconstruct the direct irradiance at both probes and assert the shadowed probe's value is strictly less than the lit probe's (per channel or luminance). AC (2) is verified by this assertion plus eyeball in-engine.

Empty/absent (no static lights) emits no section — the loader treats absence as direct = 0.

### Task 4: Renderer loader + bind for the direct atlas (`render/sh_volume.rs`, `render/mod.rs`)
Load the new section at level load and create a GPU texture + view for the direct atlas, mirroring how `ShVolumeResources::new` builds the indirect atlas (`sh_volume.rs:270`). Because the atlas is BC6H-at-rest (D8), create the texture with the `Bc6hRgbUfloat` format and upload the compressed blocks directly (hardware decode at sample time) — same as the lightmap irradiance atlas's BC6H load path; honor the uncompressed-debug format tag if present. The octahedral tile geometry is unchanged, so the bind/sampler setup is otherwise identical to the indirect atlas. Expose it on a bind group the DYNAMIC pipelines can bind WITHOUT consuming the reserved group-2 dynamic-direct slot. Preferred: add the direct atlas texture as an additional binding inside the EXISTING SH resources group (mesh group 4 / forward+billboard group 3) — `forward.wgsl` simply doesn't declare it, so the static path is unaffected and the group stays shared (BGL may carry entries a shader ignores; `research.md` confirms the precedent). When the section is absent, bind a minimal valid 4×4 BC6H zero-block texture (a single zero-block satisfies BC6H's ≥4/4-aligned rule, D8) so the dynamic shaders read direct = 0; do NOT use a 1×1 texture, which is invalid for `Bc6hRgbUfloat`. Alternatively the uncompressed-debug-tag path may be used for the fallback, but the 4×4 BC6H zero block is the default. Update visibility flags on the new BGL entry to cover the consuming stages (`FRAGMENT`, plus `COMPUTE` only if a compose variant is chosen). Honor the §10/§9 group budget — no new group if the shared SH group has room.

### Task 5: Entity + billboard shaders sample indirect + direct (`skinned_mesh.wgsl`, `billboard.wgsl`)
Declare the direct atlas binding (Task 4 slot) and append the SAME `sh_sample.wgsl` helper read against it (the helper is binding-agnostic — §8 shader composition). In `skinned_mesh.wgsl` `fs_main`, compute `direct = sample_sh_direct(world_pos, n, n)` alongside the existing `indirect` and return `base_color.rgb * (indirect + direct)`. Mirror in `billboard.wgsl` `fs_main` — but it is a SIBLING edit, NOT a copy-paste: the billboard SH read CONFIRMED-DIFFERS from `skinned_mesh.wgsl` in three concrete ways (verified against source; see `research.md` "Billboard shader SH read"). (a) Billboard's `sample_sh_indirect(world_pos, normal)` (`billboard.wgsl:264`) takes a SINGLE normal arg and forwards it to both the shading- and geo-normal slots with `reject_backface = false`; the caller passes `N = V` = camera-forward (`billboard.wgsl:295-299`) because a camera-facing sprite has no stable surface normal. So the billboard direct sample must pass camera-forward `N = V` as its normal, not a per-vertex geometric normal — add a single-normal `sample_sh_direct` matching that convention. (b) Billboards bind `sh_total_atlas` at group 3 b1 (`billboard.wgsl:73`), the SHARED group-3 SH layout (alongside `forward.wgsl`), NOT mesh group 4 — the direct binding goes in the group-3 SH layout for billboards. (c) The direct sample sums into the lighting term after the existing `sh_ambient` read, before the albedo/opacity multiply (~`billboard.wgsl:389-390`). The two full octahedral probe loops (indirect + direct, each an 8-probe weighted evaluation) are the runtime cost (D2). Keep `reject_backface = false` and Chebyshev on for the direct sample, matching the dynamic-object indirect precedent (entities are not static surfaces). `forward.wgsl` is NOT edited.

## Wire format

`DirectShVolumeSection` mirrors `OctahedralShVolumeSection` exactly except it carries direct (not indirect) coefficients and NO depth moments / animation data (direct is static and dense): little-endian; the same grid header (dims, origin, cell-size); a dense per-probe octahedral tile block in x-fastest linear probe order; per-probe tile byte layout BYTE-IDENTICAL to the indirect atlas tile (the runtime sampler is shared). The atlas payload is BC6H-compressed at rest (D8) with an irradiance-format tag (`IRRADIANCE_FORMAT_BC6H`, plus the uncompressed-debug variant) carried in the section header exactly as the lightmap section (`lightmap_bake.rs:177-191`) — BC6H re-encodes only the texel block payload, so the LOGICAL tile geometry above is unchanged. Empty-list / no-static-lights encoding: the section is omitted entirely (loader treats absence as direct = 0), matching how `DeltaShVolumes` is `Option` and skipped when no animated lights exist. State explicitly in the implementation which existing section the byte layout mirrors.

## Resolved decisions

Mechanical findings from the draft review are resolved, and the two architectural questions were settled by the owner (below). The spec is ready for promotion.

- **Lobe form / normalization → VALIDATION-GATED (D7, Task 0).** The D3 convention stays the design but is empirically confirmed FIRST by the one-probe/one-light A/B parity harness, which is the source of truth. If the harness shows the lobe/normalization is wrong, D3 is corrected from its findings before broad work. Harness pass is an acceptance criterion; the dense bake (Task 2) depends on it in Sequencing (Phase 0 gate).
- **Storage doubling → BC6H-at-rest IN v1 (D8, Tasks 1/3/4).** The direct atlas is stored BC6H-compressed at rest, reusing the lightmap irradiance atlas's encoder (`encode_bc6h_rgb_from_f32_rgba`, `bc6h.rs:71`; emit path `CompositedAtlas::encode_section`, `lightmap_bake.rs:172`/`:184`) and the hardware `Bc6hRgbUfloat` decode at load. Tile geometry is unchanged (sampler math untouched). Footprint is reported post-compression. AC added.
- **Bake-time cost → per-probe light-reach culling IN v1 (D9, Task 2).** The direct bake culls non-reaching static lights per probe before the shadow-ray pass, using the delta bake's two-stage reach test (`light_aabb` `affinity_grid.rs:274` / `reachable_leaves` `affinity_grid.rs:156` via `decompose_affinity` `affinity_grid.rs:95`), inside the D6 determinism contract. Baked output is byte-identical to an unculled bake (culling skips only provably-zero contributions); bake-time improvement reported, mirroring `log_per_light_culling` (`delta_sh_bake.rs:377`). AC added.
- **Billboard shader parity → CONFIRMED DIFFERENT (Task 5/6).** Opened `billboard.wgsl`: its `sample_sh_indirect` (`billboard.wgsl:264`) takes a SINGLE normal arg with `reject_backface = false` and the caller passes camera-forward `N = V` (`billboard.wgsl:295-299`); it binds the shared group-3 SH layout (b1, `billboard.wgsl:73`), NOT mesh group 4; and it already reads a richer group-0 `Uniforms` carrying `lighting_isolation` (`billboard.wgsl:9-19`). So the billboard edit is a sibling with three concrete deviations from the mesh edit (camera-forward normal, group-3 binding, isolation rides the existing uniform) — captured in Tasks 5 and 6. Not a copy-paste of the mesh change.
- **Where the direct binding lives → shared SH group (Task 4).** The direct atlas binds inside the EXISTING SH resources group (mesh group 4 / forward+billboard group 3); `forward.wgsl` simply doesn't declare it (the billboard's already-declared-but-unread animated bindings, `billboard.wgsl:77-81`, confirm a shared BGL may carry entries a shader ignores). Group 2 (dynamic-direct) stays free. Visibility flags cover only the consuming stages.

### Architectural questions — RESOLVED (owner, 2026-06)

1. **Principle reconciliation → RESOLVED; the over-reaching guideline was removed.** The old "direct illumination is fully dynamic / every light reaches direct through exactly one technique / they never re-weight each other" language mistook an **authoring** concern for an **engine** concern. It was revised in the context library: `index.md` §2 ("Baked over computed") and `rendering_pipeline.md` §4 now state that static-vs-dynamic is an authoring choice, and the actual engine invariant is narrower — a physical light's contribution must not be **double-counted on a given receiver** (overlapping static + dynamic must not over-brighten the same fragment); a single light MAY bake into more than one layer when those layers serve different receivers (static surfaces via the lightmap, movers via this separate baked layer). This spec already honors that invariant by construction (D1: separate atlas, `forward.wgsl` untouched, no fragment sums the same light twice). No remaining deviation.

2. **Future group-2 dynamic-direct double-count → PINNED (D10).** Settled as an invariant; see D10.
